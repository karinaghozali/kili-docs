    Format de la donnée

    Le type d'export par défaut est un JSON, et le format est celui de l'API Google.

    La fonctionnalité permet d'exporter l'intégralité de la donnée de Kili Technology.

    L'export est une liste d'assets. Chaque asset est structuré de la manière suivante :

    • id: Identifiant unique de l'Asset dans la base de donnée de Kili

    • content: Contenu de l'Asset ou URL vers l'Asset.

    • createdAt: Date de création de l'Asset.

    • externalId: External ID de l'Asset, renseigné au chargement de l'Asset.

    • instructions: Liste des annotation de la labélisation. Voir détail plus bas.

    • isHoneypot: Indique si cet asset est un Honeypot.

    • honeypotMark: Si cet Asset est un Honeypot, moyenne des honeypots obtenus par les annotateurs sur cet Asset. Nul sinon.

    • isUsedForConsensus: dans la cas de l'utilisation du consensus comme indicateur de qualité pour le projet, ce champ permet d'indiquer que l'asset est retenu pour être annoté par n annotateurs.

    • consensusMark: dans la cas d'Asset utilisé pour le consensus, note d'accord entre les annotateurs

    • isInstructions: Indique si cet Asset fait partie des instructions.

    • status: Statut de l'asset. Peut prendre les valeurs : TODO, ONGOING, LABELED, REVIEWED

    • updatedAt: Date de la dernière modification de l'Asset

    • labels: La liste des labels de l'asset: chaque label est structuré de la manière suivante

      • id: Identifiant unique du Label dans la base de donnée de Kili

      • createdAt: date de création du Label

      • honeypotMark: dans la cas d'Asset de type Honeypot, note d'accord entre la labélisation et la cible. Dans la cas d'un Asset qui n'est pas de type Honeypot, le champ est nul.

      • jsonResponse: Liste des annotation de la labélisation. Voir détail plus bas.

      • labelType: Type de label permettant d'identifier si le label est une prédiction (réalisée par un modèle), un label ordinaire (réalisé par un annotateur) ou une revue (réalisée par un reviewer). Peut prendre les valeurs suivantes : PREDICTION, DEFAULT, REVIEW

      • totalSecondsToLabel: Durée de la labélisation, exprimée en s.

    • author: Personne qui a créé le Label

      • id: ID dans Kili de l'auteur du Label

      • email: Nom de l'auteur du Label

      • name: Adresse email de l'auteur du Label

    jsonResponse est une chaîne de caractères JSON qui contient la liste des annotations. Les annotations Kili suivent de près le format Google :

    • pour l'image : https://cloud.google.com/vision/docs/detecting-logos

    • pour la reconnaissance d'entités nommées : https://cloud.google.com/natural-language/docs/analyzing-entities

    Image

    • id: Identifiant unique du Label dans le système de donnée Kili

    • score: Lorsqu'un modèle de pré-annotation est utilisé, score de confiance de la détection d'objet

    • description: Nom de l'objet

    • isOccluded: Variable booléenne indiquant si l'objet est obstrué

    • isCut: Variable booléenne indiquant si l'objet est coupé par un bord de l'image

    • boundingPoly: polygone du contour de l'objet

      • vertices: liste des sommets du polygone. Dans le cas d'une bounding box, 4 sommets

        • x: abscisse de la position du sommet depuis top left et exprimé en pourcentage de la taille totale de l'image

        • y: ordonnée de la position du sommet depuis top left et exprimé en pourcentage de la taille totale de l'image

    NER

    • text: Contenu du document vu par l'interface / le modèle de NER

    • entities: Liste des entités, comprenant les attributs suivants

      • name: Nom de l'entité

      • salience: Score de salience de l'entitié, si fourni par le modèle sous-jacent. 1 dans le cas d'une annotation manuelle.

      • type: Type de l'entité.

      • mentions: Liste des endroits où l'entité est mentionnée dans le texte

        • id: id de la mention

        • text:

          • beginOffset: position de la mention de l'entité

          • content: contenu de la mention

        • type: Type de la mention de l'entité. Aujourd'hui, est égal au type de l'entité

        • groups: Dans le cas d d'une annotation avec regroupement des entités, liste des noms groupes auquel appartient d'entité.
