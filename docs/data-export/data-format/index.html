    Data format

    The default export type is a JSON, and the format is that of the API Google.

    The functionality allows you to export all the data from Kili Technology.

    The export is a list of assets. Each asset is structured from the as follows:

    • id: Unique identifier of the Asset in the Kili database

    • content: Content of the Asset or URL to the Asset.

    • createdAt: Date of creation of the Asset.

    • externalId: External ID of the Asset, filled in when loading Asset.

    • isHoneypot: Indicates whether this asset is a Honeypot.

    • honeypotMark: If this Asset is a Honeypot, average of the honeypots obtained by the annotators on this Asset. No one else.

    • isUsedForConsensus: in the case of the use of consensus as a quality indicator for the project, this field indicates that the asset is retained to be annotated by n annotators.

    • consensusMark: in the case of Asset used for consensus, agreement note between the annotators

    • status: Status of the asset. Can take the values: TODO, ONGOING, LABELED, REVIEWED

    • updatedAt: Date of the last modification of the Asset

    • skipped: A boolean indicating whether the asset was skipped or not

    • labels: The list of asset labels: each label is structured as follows

      • id: Unique identifier of the Label in the Kili database

      • createdAt: date of creation of the Label

      • honeypotMark: in the case of Honeypot Asset, chord note between the labeling and the target. In the case of an Asset that is not no Honeypot type, the field is null.

      • jsonResponse: List of label annotations. See detail lower.

      • labelType: Type of label used to identify if the label is a prediction (made by a model), an ordinary label (made by an annotator) or a journal (carried out by a reviewer). Can take the following values: PREDICTION, DEFAULT, REVIEW

      • totalSecondsToLabel: Duration of the labeling, expressed in s.

    • author: Person who created the Label

      • id: ID in Kili of the author of the Label

      • email: Name of the author of the Label

      • name: Email address of the author of the Label

    jsonResponse is a JSON character string that contains the list of annotations. Kili annotations closely follow the Google format:

    • for the image: https://cloud.google.com/vision/docs/detecting-logos

    • for the recognition of named entities: https://cloud.google.com/natural-language/docs/analyzing-entities

    Image

    Image Classification (single-class)

    • categories: List of categories
      • name: Name of the category
      • confidence: Confidence (100 by default when done by human)

    Image Classification (multi-class)

    • categories: List of categories
      • name: Name of the category
      • confidence: Confidence (100 by default when done by human)

    Image Object Detection (bounding box)

    • annotations:
      • score: When a pre-annotation model is used, the score is confidence in object detection
      • description: Name of the object
      • boundingPoly: Polygon of the object contour
        • normalizedVertices: List of vertices of the polygon. In the case of a bounding box, 4 vertices
          • x: Abscissa of the vertex position from top left and expressed as a percentage of the total image size
          • y: Ordinate of the vertex position from top left and expressed as a percentage of the total image size

    Image Object Detection (bounding box + polygon)

    Same as Image Object Detection (bounding box).

    Image Object Detection (semantic)

    Same as Image Object Detection (bounding box).

    Optical Character Recognition

    • annotations:
      • score: When a pre-annotation model is used, the score is confidence in object detection
      • description: Name of the object
      • boundingPoly: Polygon of the object contour
        • normalizedVertices: List of vertices of the polygon. In the case of a bounding box, 4 vertices
          • x: Abscissa of the vertex position from top left and expressed as a percentage of the total image size
          • y: Ordinate of the vertex position from top left and expressed as a percentage of the total image size
      • text: Content of the text recognized by the OCR

    Text

    Text Classification (single-class)

    • categories: List of categories
      • name: Name of the category
      • confidence: Confidence (100 by default when done by human)

    Text Classification (multi-class)

    • categories: List of categories
      • name: Name of the category
      • confidence: Confidence (100 by default when done by human)

    Text Named-Entity Recognition

    • text: Document content seen by the interface / NER model

    • entities: List of entities, including the following attributes

      • name: Name of the entity

      • salience: The salience score of the entity, if provided by the underlying model. 1 in the case of manual annotation.

      • type: Type of the entity.

      • mentions: List of places where the entity is mentioned in the text

        • id: id of the mention

        • text:

          • beginOffset: position of the entity mention

          • content: content of the mention

        • type: Type of the entity mention. Today, is equal the type of the entity
